# ADR-003: 整合既有 VLM-RAG

## 狀態
已採用

## 背景
本專案需具備高效的視覺語言檢索能力，能根據用戶查詢快速定位相關影片片段與語意摘要。既有 VLM-RAG 系統已在安防領域驗證，具備穩定 API 與高檢索品質。

## 決策
選擇直接整合現有 VLM-RAG 系統作為檢索後端。

## 理由
- 已有穩定部署與 API 文件
- 檢索品質高，支援多種查詢型態
- 可快速對接，降低開發成本
- 支援擴展與自訂

## 後果
- 可快速實現檢索功能
- 需維護與既有 VLM-RAG 系統的 API 相容性
- 依賴外部服務穩定性

## 替代方案
- 自建檢索系統：彈性高但開發成本大
- 使用開源多模態檢索框架（如 CLIP-Retrieval）：需額外部署與調整
- 直接用 LLM 生成：檢索精度較低
