# Phase 4 – UI 架構與互動流程設計（@ARCH）

## 1. UI 畫面結構與區塊說明
- 查詢輸入區：用戶輸入自然語言問題
- 事件列表區：顯示檢索到的事件摘要（含時間、描述、分數）
- AI 回答區：顯示 LLM 生成的答案
- 回饋區：用戶可點選「滿意/不滿意」並輸入意見
- 錯誤提示區：顯示 API/系統錯誤訊息

## 2. 互動流程與狀態流轉描述
1. 用戶輸入查詢並送出
2. 前端呼叫 VQA API，取得事件列表與 AI 回答
3. 顯示事件摘要與 AI 回答
4. 用戶可對本次回答給予「滿意/不滿意」回饋
5. 回饋結果送至後端記錄
6. 若 API 失敗，顯示錯誤提示，允許重試

## 3. 前後端資料流設計
- 前端送出查詢（POST /query），後端回傳事件列表與 AI 回答
- 回饋（POST /feedback）包含查詢內容、事件ID、AI回答、用戶回饋
- 所有 API 皆需處理異常並回傳錯誤訊息

## 4. Checkpoint 1 審查重點清單
- 是否涵蓋主要使用情境（如事件查詢、AI 回答、回饋）
- UI 結構與互動流程是否合理、可實作
- 前後端資料流設計是否清晰
- 錯誤處理與回饋流程是否明確
- 畫面區塊分工與狀態流轉是否易於維護
